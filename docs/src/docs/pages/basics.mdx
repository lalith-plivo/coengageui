# Learn the basics

This guide will help you get familiar with core CoengageUI concepts.
Please read this guide and **theming** section before starting development
to learn about all available theming features and components props.

## Getting help

CoengageUI has a very friendly community, we are always happy to help you get started:

- [Join the Discord community](https://discord.gg/eUZpPbpxb4) – it is the easiest way to get help, all questions are usually answered in about 30 minutes.
- [GitHub Discussions](https://github.com/coengagedev/coengage/discussions) – ask anything about the project or give feedback.

## Using documentation

CoengageUI documentation includes more than 150 pages, to use it efficiently remember 2 keyboard shortcuts:

- `⌘ + K` or `Ctrl + K` – focus search field, searching components and hooks is the best way to jump straight to the page you are looking for.
- `⌘ + J` or `Ctrl + J` – toggle color scheme (light/dark). All components support both light and dark color schemes, using this shortcut is the easiest way to preview both color schemes.

## CoengageUI packages

- `@coengageui/hooks` – collection of 30+ react hooks for state and UI management.
- `@coengageui/styles` – [emotion](https://emotion.sh/) based css-in-js library that is used in all CoengageUI components. Usually this package is installed automatically and exported from `@coengageui/core` – there is no need to install it separately, [learn more about createStyles here](/theming/create-styles/).
- `@coengageui/core` – core components library – 100+ components, exports everything from `@coengageui/styles`.
- `@coengageui/notifications` – a fully featured [notifications system](/others/notifications/).
- `@coengageui/prism` – [code highlight](/others/prism/) built with [prism-react-renderer](https://github.com/FormidableLabs/prism-react-renderer).
- `@coengageui/rte` – a Quill based [rich text editor](/others/rte/).
- `@coengageui/dropzone` – manages [files drag 'n' drop](/others/dropzone/) of files to an area or the entire screen.
- `@coengageui/ssr` – [server side rendering](/theming/ssr/) utilities.
- `@coengageui/next` – components and ssr utils for [Next.js integration](/theming/next/).
- `gatsby-plugin-coengage` – [Gatsby plugin](/theming/gatsby/) to setup ssr.
- `@coengageui/eslint-config` – ESLint and Prettier configuration that the CoengageUI project uses.

## Theming

CoengageUI theming supports changing colors, spacing, box-shadows, font families, font sizes and many other properties.
To configure theme, wrap your app with a [CoengageUIProvider](/theming/coengageui-provider/) component:

```tsx
import { CoengageUIProvider } from '@coengageui/core';

function App() {
  return (
    <CoengageUIProvider
      theme={{
        // Override any other properties from default theme
        fontFamily: 'Open Sans, sans serif',
        spacing: { xs: 15, sm: 20, md: 25, lg: 30, xl: 40 },
      }}
    >
      <YourApp />
    </CoengageUIProvider>
  );
}
```

Learn more about [CoengageUIProvider](/theming/coengageui-provider/) and [extending theme](/theming/extend-theme/).

## Dark color scheme

All CoengageUI components support light and dark color scheme out of the box.
You can edit the details of each color scheme via [CoengageUIProvider](/theming/coengageui-provider/):

```tsx
import { CoengageUIProvider } from '@coengageui/core';

function App() {
  return (
    <CoengageUIProvider theme={{ colorScheme: 'dark' }}>
      <YourApp />
    </CoengageUIProvider>
  );
}
```

To learn how to implement color scheme changes via context follow [dark theme guide](/theming/dark-theme/).

## Components props

### Shared props

All CoengageUI components support these props:

- `className` – adds class to root element.
- `style` – adds style to root element.
- Margin props:
  - `m` – sets `margin` property on root element.
  - `my` – sets `margin-top` and `margin-bottom` properties on root element.
  - `mx` – sets `margin-right` and `margin-left` properties on root element.
  - `mt` – sets `margin-top` property on root element.
  - `mb` – sets `margin-bottom` property on root element.
  - `ml` – sets `margin-left` property on root element.
  - `mr` – sets `margin-right` property on root element.
- Padding props:
  - `p` – sets `padding` property on root element
  - `py` – sets `padding-top` and `padding-bottom` properties on root element
  - `px` – sets `padding-right` and `padding-left` properties on root element
  - `pt` – sets `padding-top` property on root element
  - `pb` – sets `padding-bottom` property on root element
  - `pl` – sets `padding-left` property on root element
  - `pr` – sets `padding-right` property on root element

```tsx
import { Button } from '@coengageui/core';

function Demo() {
  return (
    <Button className="my-button" style={{ backgroundColor: '#fff' }} mx={20}>
      My button
    </Button>
  );
}
```

Margin props can get value from `theme.spacing`:

```tsx
<Button mt="xl" /> // -> margin-top: theme.spacing.xl
<Button ml="-md" /> // -> margin-left: theme.spacing.md * -1
<Button m={20} /> // -> margin: 20px
```

Same is applied to padding props:

```tsx
<Paper p="xl" /> // -> padding: theme.spacing.xl
<Container px={30} /> // -> padding-left: 30px, padding-right: 30px
```

### Color prop

CoengageUI components work with colors defined in [theme.colors](/theming/extend-theme/#default-colors).
`theme.colors` is an object that contains an array of 10 shades per each color. To use predefined colors in components set `color` prop:

```tsx
<Badge color="teal" />
<Button color="violet" />
```

You can [extend theme](/theming/extend-theme/) with any amount of your colors:

```tsx
import { CoengageUIProvider } from '@coengageui/core';

function App() {
  return (
    <CoengageUIProvider
      theme={{
        // Theme is deeply merged with default theme
        colors: {
          // Add your color
          'deep-blue': ['#E9EDFC', '#C1CCF6', '#99ABF0' /* 7 other shades */],
          // or replace default theme color
          blue: ['#E9EDFC', '#C1CCF6', '#99ABF0' /* 7 other shades */],
        },
      }}
    >
      <YourApp />
    </CoengageUIProvider>
  );
}
```

Note that component appearance usually depends on `variant` prop and current `theme.colorScheme`.

### Sizes

Most CoengageUI components support `size` prop with xs, sm, md, lg and xl values:

```tsx
<Button size="xl" />
<Badge size="xs" />
```

The `size` prop controls various css properties across all supported components.
In some components where size is associated with only one value, you can set it in px:

```tsx
<Slider size="xs" /> // Predefined xs size
<Slider size={20} /> // -> 20px track height, other parts are scaled from this value
```

### Spacing and padding

Components that have padding get values from [theme.spacing](/theming/extend-theme/#spacing-radius-and-shadows),
default values are:

```tsx
{ xs: 10, sm: 12, md: 16, lg: 20, xl: 24 }
```

To change these values set `spacing` property on [theme](/theming/extend-theme/):

```tsx
import { CoengageUIProvider } from '@coengageui/core';

function App() {
  return (
    <CoengageUIProvider theme={{ spacing: { xs: 15, sm: 20, md: 25, lg: 30, xl: 40 } }}>
      <YourApp />
    </CoengageUIProvider>
  );
}
```

Later when you use CoengageUI components you can reference these values in `spacing` or other similar props or set the spacing in px:

```tsx
<Group spacing="md" /> // -> md spacing from theme.spacing
<Group spacing={40} /> // 40px spacing
```

### Shadows

Components that use the box-shadow property get values from [theme.shadows](/theming/extend-theme/#spacing-radius-and-shadows),
default values are:

```tsx
{
  xs: '0 1px 3px rgba(0, 0, 0, 0.05), 0 1px 2px rgba(0, 0, 0, 0.1)',
  sm: '0 1px 3px rgba(0, 0, 0, 0.05), rgba(0, 0, 0, 0.05) 0px 10px 15px -5px, rgba(0, 0, 0, 0.04) 0px 7px 7px -5px',
  md: '0 1px 3px rgba(0, 0, 0, 0.05), rgba(0, 0, 0, 0.05) 0px 20px 25px -5px, rgba(0, 0, 0, 0.04) 0px 10px 10px -5px',
  lg: '0 1px 3px rgba(0, 0, 0, 0.05), rgba(0, 0, 0, 0.05) 0px 28px 23px -7px, rgba(0, 0, 0, 0.04) 0px 12px 12px -7px',
  xl: '0 1px 3px rgba(0, 0, 0, 0.05), rgba(0, 0, 0, 0.05) 0px 36px 28px -7px, rgba(0, 0, 0, 0.04) 0px 17px 17px -7px',
}
```

To change these values set the `shadows` property on [theme](/theming/extend-theme/):

```tsx
import { CoengageUIProvider } from '@coengageui/core';

function App() {
  return (
    <CoengageUIProvider
      theme={{
        shadows: {
          xs: '1px 1px 1px rgba(0, 0, 0, 0.3)',
          sm: '1px 1px 4px rgba(0, 0, 0, 0.4)',
          md: '3px 3px 4px rgba(0, 0, 0, 0.4)',
          lg: '3px 3px 4px 5px rgba(0, 0, 0, 0.4)',
          xl: '3px 3px 4px 15px rgba(0, 0, 0, 0.4)',
        },
      }}
    >
      <YourApp />
    </CoengageUIProvider>
  );
}
```

Later when you use CoengageUI components you can reference these values in `shadow` prop or define your own shadow:

```tsx
<Paper shadow="xl" /> // -> xl shadow from theme.shadows
<Paper shadow="1px 3px 4px rgba(0, 0, 0, 0.4)" /> // -> your own shadow not related to theme.shadows
```

## Polymorphic components

Some CoengageUI components support changing root element with `component` prop,
you can pass html element or React component to the prop:

```tsx
import { Text, Button } from '@coengageui/core';
import { Link } from 'react-router-dom';

function Demo() {
  return (
    <>
      {/* with html element */}
      <Text component="a" href="https://coengage.dev">
        My link text
      </Text>

      {/* with react-router-dom Link */}
      <Button component={Link} to="/hello">
        My link text
      </Button>
    </>
  );
}
```

To use CoengageUI components with Next.js Link:

- set `component` prop to `a`
- wrap it with Link

```tsx
import { Button } from '@coengageui/core';
import Link from 'next/link';

function Demo() {
  return (
    <Link href="/hello" passHref>
      <Button component="a">Next link button</Button>
    </Link>
  );
}
```

If you use polymorphic with TypeScript you will also need to set type for full types coverage,
otherwise you will have untyped event handlers and ref props:

```tsx
import { Button } from '@coengageui/core';
import { Link } from 'react-router-dom';

function Demo() {
  return (
    <>
      <Button<'a'> component="a" href="#">
        Anchor button
      </Button>

      <Button<typeof Link> component={Link} to="/hello">
        React router button
      </Button>
    </>
  );
}
```

## Getting element ref

You can get ref of most components with their `ref` prop:

```tsx
import { useRef } from 'react';
import { Button, Paper, TextInput } from '@coengageui/core';

function Demo() {
  const buttonRef = useRef<HTMLButtonElement>();
  const paperRef = useRef<HTMLDivElement>();
  const inputRef = useRef<HTMLInputElement>();

  return (
    <>
      <Button ref={buttonRef} />
      <Paper ref={paperRef} />
      <TextInput ref={inputRef} />
    </>
  );
}
```

## TypeScript

### Exported types

@coengageui/core package export types to help you build components and styles with TypeScript:

- `CoengageUITheme` – theme interface defined in CoengageUIProvider.
- `ColorScheme` – union of `'light' | 'dark'`.
- `CoengageUIColor` – union of all default colors, also accepts any string.
- `CoengageUIGradient` – gradient interface used in [Button](/core/button/), [ThemeIcon](/core/theme-icon/) and other components.
- `CoengageUIShadow` – union of all default shadows.
- `CoengageUISize` – union of `'xs' | 'sm' | 'md' | 'lg' | 'xl'`.
- `CoengageUINumberSize` – union of `CoengageUISize | number`.

### Components props

You can import props type of any component by adding `Props` to the component name:

```tsx
import type { ButtonProps } from '@coengageui/core';
```

### Polymorphic components types

@coengageui/core package exports `PolymorphicRef` and `PolymorphicComponentProps` types to help you build
custom components with ref forwarding:

```tsx
import { forwardRef } from 'react';
import { PolymorphicRef, PolymorphicComponentProps } from '@coengageui/core';
import { Link } from 'react-router-dom';

// Define all component specific props here
interface _TextProps {
  bold?: boolean;
}

type TextProps<C extends React.ElementType> = PolymorphicComponentProps<C, _TextProps>;

type TextComponent = <C extends React.ElementType = 'div'>(
  props: TextProps<C>
) => React.ReactElement;

const Text: TextComponent = forwardRef(
  <C extends React.ElementType = 'div'>(
    { bold, component, ...others }: TextProps<C>,
    ref: PolymorphicRef<C>
  ) => {
    const Element = component || 'div';
    return <Element style={{ fontWeight: bold ? 700 : 400 }} ref={ref} {...others} />;
  }
);

function Demo() {
  return (
    <>
      {/* Default div text, props will have type React.ComponentPropsWithoutRef<'div'> & _TextProps */}
      <Text bold>Bold text</Text>

      {/* Text as an anchor, props will have type React.ComponentPropsWithoutRef<'a'> & _TextProps */}
      <Text<'a'> component="a" href="https://coengage.dev">
        CoengageUI website
      </Text>

      {/* Text as component, props will have type React.ComponentPropsWithoutRef<typeof Link> & _TextProps */}
      <Text<typeof Link> component={Link} to="/route">
        Router link
      </Text>
    </>
  );
}
```
