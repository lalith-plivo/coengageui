## use-form hook improvements

- [use-form](https://coengage.dev/hooks/use-form/) hook now includes new `getInputProps` handler to simplify inputs

- [use-form](https://coengage.dev/hooks/use-form/) hook now supports optional error messages (hook will work as before if messages are not provided):

```tsx
const form = useForm({
  initialValues: { name: '', age: 0 },
  validationRules: {
    name: (value) => value.trim().length >= 2,
    age: (value) => value > 21,
  },
  errorMessages: {
    name: 'Name must include at least 2 characters',
    age: 'You must be 21 or older to enter',
  },
});

form.validate();
form.errors;
// -> { name: 'Name must include at least 2 characters', age: 'You must be 21 or older to enter' }

form.setFieldValue('name', 'John');
form.validate();
form.errors;
// -> { name: null, age: 'You must be 21 or older to enter' }
```

## Other changes

- Components that use [Popper](https://coengage.dev/core/popper/) ([Select](https://coengage.dev/core/select/), [Menu](https://coengage.dev/core/menu/), [DatePicker](https://coengage.dev/dates/date-picker/), etc.) now support an option to disable rendering within portal
- [Rich text editor](https://coengage.dev/others/rte/) now has an option to [get editor ref](https://coengage.dev/others/rte/#get-editor-ref), it can be used, for example, to focus/blur editor or to extend editor with plugins
- Components that use [Popper](https://coengage.dev/core/popper/) now support rendering without [Portal](https://coengage.dev/core/portal/) by setting `withinPortal={false}`. This option is useful when you want to use [Select](https://coengage.dev/core/select/) and other components inside [Popover](https://coengage.dev/core/popover/) or other components that use [use-click-outside](https://coengage.dev/hooks/use-click-outside/).
- [Accordion](https://coengage.dev/core/accordion/) component now supports `iconSize` and `offsetIcon` props
- [Menu](https://coengage.dev/core/menu/) component now support disabled items
- You can now change default locale that is used in all `@coengageui/dates` components with `dateLocale` on [CoengageUIProvider](https://coengage.dev/theming/coengageui-provider/)

## Bug fixes

- [Collapse](https://coengage.dev/core/collapse/) component now works correctly with `transitionDuration={0}`
- [use-focus-trap](https://coengage.dev/hooks/use-focus-trap/) hook is now less restrictive, it fixes issue with [Popover](https://coengage.dev/core/popover/) and [Menu](https://coengage.dev/core/menu/) scrolling page when being closed
- Fix click outside not working with [Modal](https://coengage.dev/core/modal/) with outside overflow
- [InputWrapper](https://coengage.dev/core/input-wrapper/) label now has `display: inline-block` styles, these styles will prevent input focusing when empty area above the input is clicked
- [Select](https://coengage.dev/core/select/) component now correctly handles up and down arrow keys with selected value
- Fix incorrect opened dropdown state in [Select](https://coengage.dev/core/select/) component when search results are empty and nothing found message is not provided
- Fix incorrect enter key handling in [Autocomplete](https://coengage.dev/core/autocomplete/), [Select](https://coengage.dev/core/select/), [MultiSelect](https://coengage.dev/core/multi-select/) and [DatePicker](https://coengage.dev/core/date-picker/) components which resulted in unexpected form submits
- Fix incorrect space key handling in [Select](https://coengage.dev/core/select/) and [MultiSelect](https://coengage.dev/core/multi-select/) components
- Fix focus shifting to input when date is selected with keyboard in [DateRangePicker](https://coengage.dev/dates/date-picker/)
