## @coengageui/notifications improvements

### Events based functions

[Notifications system](https://coengage.dev/others/notifications/) functions were migrated to
[custom events](https://developer.mozilla.org/en-US/docs/Web/Events/Creating_and_triggering_events),
they are no longer depend on context and can be called in any part of application.
Old way with `useNotifications` hook will work until version 5.0 release.

`@coengageui/notifications` now exports the following functions:

- `showNotification` – adds given notification to notifications list or queue depending on current state and limit
- `updateNotification` – updates notification that was previously added to the state or queue
- `hideNotification` – removes notification with given id from notifications state and queue
- `cleanNotifications` – removes all notifications from notifications state and queue
- `cleanNotificationsQueue` – removes all notifications from queue

## classNames prop on CoengageUIProvider

Same as with `styles` you can now add classes to all components with `classNames` prop on CoengageUIProvider.
This approach is useful when you want to styles components with utility based CSS libraries.

```tsx
import { CoengageUIProvider, Button } from '@coengageui/core';

function App() {
  return (
    <CoengageUIProvider classNames={{ Button: { root: 'button-root', label: 'button-label' } }}>
      <Button>All Button components will have the classes above</Button>
    </CoengageUIProvider>
  );
}
```

## CoengageUI CSS variables

You can now set `withCSSVariables` prop on [CoengageUIProvider](https://coengage.dev/theming/coengageui-provider/) to add
[CoengageUI CSS variables](https://coengage.dev/theming/coengageui-provider/#coengageui-css-variables) from theme to `:root`.
This option is useful when you want to use CoengageUI theme values with CSS/SCSS or css-in-js libraries that do not have CoengageUI theme context.

```tsx
import { CoengageUIProvider } from '@coengageui/core';

function App() {
  return (
    <CoengageUIProvider withCSSVariables>
      <YourApp />
    </CoengageUIProvider>
  );
}
```

Then you will be able to use variables anywhere in your CSS:

```css
.my-button {
  background-color: var(--coengageui-color-blue-6);
  font-family: var(--coengageui-font-family);
  line-height: var(--coengageui-line-height);
}
```

## sx array syntax

All components now support arrays in [sx prop](https://coengage.dev/theming/sx/). It can be to simultaneously use `sx` prop and add it from props:

```tsx
import { Button, Sx } from '@coengageui/core';

interface MyButtonProps {
  sx?: Sx;
}

function MyButton({ sx }) {
  return <Button sx={[{ color: 'red' }, sx]}>My Button</Button>;
}
```

## Aside and Footer components

[AppShell](https://coengage.dev/core/app-shell/) now supports `Aside` and `Footer` components,
they work the same way as `Navbar` and `Header` component but render content on the opposite side:

<Anchor href="/app-shell-demo/" target="_blank" mb="xl" style={{ display: 'block' }}>
  <Center inline>
    <span style={{ marginRight: 7 }}>Open responsive example in new tab</span>
    <ExternalLinkIcon width={18} height={18} />
  </Center>
</Anchor>

```tsx
import React, { useState } from 'react';
import {
  AppShell,
  Navbar,
  Header,
  Footer,
  Aside,
  Text,
  MediaQuery,
  Burger,
  useCoengageUITheme,
} from '@coengageui/core';

export default function AppShellDemo() {
  const theme = useCoengageUITheme();
  const [opened, setOpened] = useState(false);
  return (
    <AppShell
      styles={{
        main: {
          background: theme.colorScheme === 'dark' ? theme.colors.dark[8] : theme.colors.gray[0],
        },
      }}
      navbarOffsetBreakpoint="sm"
      asideOffsetBreakpoint="sm"
      fixed
      navbar={
        <Navbar p="md" hiddenBreakpoint="sm" hidden={!opened} width={{ sm: 200, lg: 300 }}>
          <Text>Application navbar</Text>
        </Navbar>
      }
      aside={
        <MediaQuery smallerThan="sm" styles={{ display: 'none' }}>
          <Aside p="md" hiddenBreakpoint="sm" width={{ sm: 200, lg: 300 }}>
            <Text>Application sidebar</Text>
          </Aside>
        </MediaQuery>
      }
      footer={
        <Footer height={60} p="md">
          Application footer
        </Footer>
      }
      header={
        <Header height={70} p="md">
          <div style={{ display: 'flex', alignItems: 'center', height: '100%' }}>
            <MediaQuery largerThan="sm" styles={{ display: 'none' }}>
              <Burger
                opened={opened}
                onClick={() => setOpened((o) => !o)}
                size="sm"
                color={theme.colors.gray[6]}
                mr="xl"
              />
            </MediaQuery>

            <Text>Application header</Text>
          </div>
        </Header>
      }
    >
      <Text>Resize app to see responsive navbar in action</Text>
    </AppShell>
  );
}
```

## Other changes

- [@coengageui/spotlight](https://coengage.dev/others/spotlight/) now exports event based functions to open/close spotlight and perform other actions from anywhere in your application (not only in components)
- [Button](https://coengage.dev/core/button/), [Badge](https://coengage.dev/core/badge/), [ThemeIcon](https://coengage.dev/core/theme-icon/) and [Text](https://coengage.dev/core/text/) components now support hex color values in `gradient` prop
- [Radio](https://coengage.dev/core/radio-group/) now supports setting transition duration and icon customization via props
- [Slider and RangeSlider](https://coengage.dev/core/slider/) components now support `precision` prop
- [Menu](https://coengage.dev/core/menu/), [List](https://coengage.dev/core/list/) and [Grid](https://coengage.dev/core/grid/) components were migrated to context, associated components (`Menu.Item`, `List.Item`, `Grid.Col`, etc.) can now be wrapper with other components and fragments
- `useLocalStorageValue` hook was renamed to `useLocalStorage` (old hook is still exported from `@coengageui/hooks`)
- [AppShell](https://coengage.dev/core/app-shell/) no longer parses props of Navbar and Header components, AppShell positioning logic was migrated to context and CSS variables, Navbar and Header components can now be wrapped with other components
- NextLink component was added to `@coengageui/next` package to simplify Next.js Link usage with [Menu](https://coengage.dev/core/menu/#menuitem-as-nextjs-link)
- [Global](https://coengage.dev/theming/global-styles/#global-component) component now supports array syntax for `styles`
