## Breaking changes

- [MediaQuery](https://coengage.dev/core/media-query/) now requires passing `styles` object and does not hide child element by default
- [Styles API](https://coengage.dev/theming/styles-api/) static selectors no longer use kebab case (was `.coengageui-text-input-root`, now `.coengageui-TextInput-root`)

## Configure emotion with CoengageUIProvider

You can now set emotion cache options via CoengageUIProvider:

```tsx
<CoengageUIProvider emotionOptions={{ key: 'coengage', prepend: false }}>
  <App />
</CoengageUIProvider>
```

With these options you can change styles insertion point (from 3.1 styles are prepended to head)
and configure stylis plugins. For example, you can use [stylis-plugin-rtl](https://github.com/styled-components/stylis-plugin-rtl) to add rtl support:

```tsx
import { CoengageUIProvider } from 'coengageui_core';
import rtlPlugin from 'stylis-plugin-rtl';

function Demo() {
  return (
    <CoengageUIProvider emotionOptions={{ key: 'coengage', stylisPlugins: [rtlPlugin] }}>
      <App />
    </CoengageUIProvider>
  );
}
```

## Inline styles with emotion

`styles` prop will no longer add inline styles, instead styles will be converted to emotion format.
This means that you now can subscribe to `theme` and use nested selectors in `styles`:

```tsx
<Button
  styles={(theme) => ({
    outline: {
      '&:hover': {
        backgroundColor: theme.colorScheme === 'dark' ? theme.colors.dark[8] : theme.colors.dark[0],
      },
    },

    root: {
      height: 56,
    },
  })}
>
  My custom button
</Button>
```

## use-css hook

New use-css hook is a simpler alternative to `createStyles` function.
Hook returns `css` and `cx` functions. `css` function accepts styles object and returns class name:

```tsx
import { useCss, Button } from 'coengageui_core';

function Demo({ active }) {
  const { css, cx } = useCss();
  return (
    <Button
      className={cx(css({ backgroundColor: 'red', opacity: 1 }), { [css({ opacity: 1 })]: active })}
    >
      Custom button
    </Button>
  );
}
```

## Other changes

- [use-list-state](https://coengage.dev/hooks/use-list-state/) hook now has new `applyWhere` handler
- [use-form](https://coengage.dev/hooks/use-form/) hook now supports `values` on validation rule
- [Slider and RangeSlider](https://coengage.dev/core/slider/) components were migrated to [use-move](https://coengage.dev/hooks/use-move/) hook and now supports showing label on hover (new default behavior)
- Add better focus management to [Popover](https://coengage.dev/core/popover/) – now focus will be returned to last active element (made with [use-focus-return](https://coengage.dev/hooks/use-focus-return/) hook)
- [AvatarsGroup](https://coengage.dev/core/avatar/#avatars-group) - now supports `total` prop to explicitly override the truncated length.
- Add `onDropdownOpen` and `onDropdownClose` handlers support to [Select](https://coengage.dev/core/select/), [MultiSelect](https://coengage.dev/core/multi-select/) and [Autocomplete](https://coengage.dev/core/autocomplete/) components

## Bug fixes

- Fix incorrect focus handling in [Menu](https://coengage.dev/core/menu/), [DatePicker](https://coengage.dev/dates/date-picker/) and [DateRangePicker](https://coengage.dev/dates/date-picker/) which resulted in focus not being shifted to control/input when dropdown was closed
- Fix incorrect [MultiSelect](https://coengage.dev/core/multi-select/) creatable filtering logic when the same item can be created multiple times
- Fix layout jumping on Windows and Linux when overlay components with scroll lock are opened ([Modal](https://coengage.dev/core/modal/) and [Drawer](https://coengage.dev/core/drawer)), see [use-scroll-lock](https://coengage.dev/hooks/use-scroll-lock/) documentation to learn how to offset elements with fixed position
- Fix [DatePicker](https://coengage.dev/dates/date-picker/) dropdown not updating position when used near bottom edge of viewport
- Fix incorrect random ids generated during server side rendering in Next.js
- Prevent page from rendering twice in `coengageui_next` package
