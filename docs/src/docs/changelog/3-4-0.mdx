## @coengageui/dates improvements

[Calendar](https://coengage.dev/dates/calendar/) component was rebuilt to improved date picking experience.
Now user can change level by clicking label between next and previous controls:

<Demo data={CalendarDemos.usage} demoProps={{ toggle: true }} />

This change is applied to all components that use Calendar: [DatePicker](https://coengage.dev/dates/date-picker/), [DateRangePicker](https://coengage.dev/dates/date-picker/) and [RangeCalendar](https://coengage.dev/dates/calendar/).
There are several **breaking changes** associated with this improvement:

- All @coengageui/dates components that use Calendar no longer support `withSelect` and `yearsRange` props
- Styles API names were changed to include new calendar header

## New hooks

[use-page-leave](https://coengage.dev/hooks/use-page-leave/) hook calls given function when mouse leaves page:

<Demo data={HooksDemos.usePageLeaveDemo} />

[use-input-state](https://coengage.dev/hooks/use-input-state/) hook lets you handle state of native inputs (with event in `onChange` handler) and custom inputs (with value in `onChange` handler):

```tsx
import { useInputState } from '@coengageui/hooks';
import { TextInput, NumberInput } from '@coengageui/core';

function Demo() {
  const [stringValue, setStringValue] = useInputState('');
  const [numberValue, setNumberValue] = useInputState(0);

  return (
    <>
      <input type="text" value={stringValue} onChange={setStringValue} />
      <TextInput value={stringValue} onChange={setStringValue} />
      <NumberInput value={numberValue} onChange={setNumberValue} />
    </>
  );
}
```

## Other changes

- [Popover](https://coengage.dev/core/popover/) component now supports `width` prop to set popover body width without Styles API
- [Modal](https://coengage.dev/core/modal/) component now supports `radius` prop
- [Modals manager](https://coengage.dev/others/modals/) now supports `onClose` prop for all modals, callback is called when modal is closed no matter the reason

## 3.3.0 – 3.4.0 bug fixes

- Fix missing support for margin props in [UnstyledButton](https://coengage.dev/core/button/#unstyled-button) component
- Fix incorrect initial month displayed for controlled free form [DatePicker](https://coengage.dev/dates/date-picker/)
- Fix incorrect [use-form](https://coengage.dev/hooks/use-form/) `getInputProps` type
- Fix incorrect [Drawer](https://coengage.dev/core/drawer/) default z-index property
- Fix incorrect value set n hidden input in [Select](https://coengage.dev/core/select/) component
- Fix issue when it was not possible to delete last character in controlled searchable [Select](https://coengage.dev/core/select/) component
- Fix items filtering logic for searchable [Select](https://coengage.dev/core/select/) – now items state can be fully controlled with filter function
- Fix [Group](https://coengage.dev/core/group/) component styles with column direction and apart position
- Fix [Select](https://coengage.dev/core/select/), [MultiSelect](https://coengage.dev/core/multi-select/) and [Autocomplete](https://coengage.dev/core/autocomplete/) dropdowns positioning when components are out of viewport
- Fix [Popper](https://coengage.dev/core/popper/) component not being unmounted when reference element is removed from the dom
- Fix scrollbars displaying under [Modal](https://coengage.dev/core/modal/) overlay for modals with outside overflow
- Remove unexpected scrollbar from [TransferList](https://coengage.dev/core/transfer-list/), [Select](https://coengage.dev/core/select/) and [MultiSelect](https://coengage.dev/core/multi-select/) components
