## Other changes

- [Center](https://coengage.dev/core/center/) component now supports `inline` prop to center inline elements
- Initial highlighted item in [Menu](https://coengage.dev/core/menu/) component no longer set on menu open â€“ user will have to use up or down arrow at least one to make selection visible
- [Card](https://coengage.dev/core/card/), [Paper](https://coengage.dev/core/paper/) and [Avatar](https://coengage.dev/core/avatar/) components now support setting custom component as root element via `component` prop
- [Text](https://coengage.dev/core/text/#dimmed) component now supports special `dimmed` color and `inline` prop to set line-height to 1
- [Select](https://coengage.dev/core/select/), [Autocomplete](https://coengage.dev/core/autocomplete/) and [MultiSelect](https://coengage.dev/core/multi-select/) dropdowns are closed when input is clicked
- Components that support changing root element with `component` prop ([Button](https://coengage.dev/core/button/), [ActionIcon](https://coengage.dev/core/action-icon/), etc.) now have better TypeScript support
- [Loader](https://coengage.dev/core/loader/) component now supports 2 new variants
- [Select](https://coengage.dev/core/select/) component now supports `onSearchChange` event which allows to change data based on search query
- Indeterminate and checked icons were replaced in [Checkbox](https://coengage.dev/core/checkbox/) to make component more recognizable
- [Menu](https://coengage.dev/core/menu/), [Accordion](https://coengage.dev/core/accordion/) and [Card](https://coengage.dev/core/card/) components now support simpler imports, e.g. `<Menu.Item />` instead of separate `<MenuItem />` import
- [NativeSelect](https://coengage.dev/core/native-select/#right-section), [Select](https://coengage.dev/core/select/#right-section) and [MultiSelect](https://coengage.dev/core/multi-select/#right-section) components now support icon in right section changing
- [Modal](https://coengage.dev/core/modal/) and [Drawer](https://coengage.dev/core/drawer/) components now return focus to last active element after being closed
- [Overlay](https://coengage.dev/core/overlay/#with-gradient) component now supports setting gradient instead of background color
