## Breaking changes

- CoengageUI was migrated to [emotion](https://emotion.sh/) from react-jss and no longer supports react-jss
- `@coengageui/core` package no longer exports react-jss `theming` context, use [createStyles](https://coengage.dev/theming/create-styles/) instead
- All components no longer support `themeOverride` prop due to performance reasons
- `elementRef` prop was replaced with `ref` in all components
- Default line-height (`theme.lineHeight`) was increased from 1.4 to 1.55
- Default [Container](https://coengage.dev/core/container/) sizes were changed
- [Divider](https://coengage.dev/core/divider/) component no longer supports `margins` prop, use `mx` or `my` instead
- `createStyles` function now returns an object of `{ classes, cx }` instead of just `classes`

## Migration to emotion

CoengageUI no longer uses react-jss, all components were migrated to [emotion](https://emotion.sh/) based css-in-js library
exposed as separate `@coengageui/styles` package and as a part of `@coengageui/core`. This means that you will have to:

- migrate all of your custom styles built with react-jss to new `createStyles` hook creator
- setup new strategy for server side rendering

### createStyles

[Read createStyles documentation](https://coengage.dev/theming/create-styles/)

From version 3.0 [createStyles](https://coengage.dev/theming/create-styles/) is the main way to add styles to CoengageUI components. Core features:

- API is similar to react-jss
- As fast and lightweight as emotion: `createStyles` extends [@emotion/react](https://emotion.sh/docs/@emotion/react)
- Subscribes to your CoengageUI theming context
- Supports all emotion features: automatic critical css extraction during server side rendering, lazy evaluation, dynamic theming, etc.
- Server side rendering support: Next.js, Gatsby or any other environment
- Fully featured TypeScript support

Basic `createStyles` usage example:

<Demo data={CreateStylesDemos.createStylesBasics} />

### Server side rendering

Since CoengageUI now uses emotion instead of react-jss you will need to change server side rendering strategy.
CoengageUI now has 3 new packages to help you setup server side rendering:

- `@coengageui/ssr` – general server side rendering utilities
- `@coengageui/next` – Next.js specific ssr configurations
- `gatsby-plugin-coengage` – enable ssr in Gatsby

Follow these guides to get started with ssr:

- [SSR with Next.js](https://coengage.dev/theming/next/)
- [SSR with Gatsby.js](https://coengage.dev/theming/gatsby/)
- [General approach to SSR](https://coengage.dev/theming/ssr/)

## Ref forwarding

All components that previously used `elementRef` were rebuilt with `React.forwardRef`, use `ref` prop to access element ref:

```tsx
<TextInput ref={(node) => {}} />
```

## Unique ids during ssr

It's no longer required to set static ids on [Grid](https://coengage.dev/core/grid/), [Menu](https://coengage.dev/core/menu/), all inputs and some other components
if you wrap your application in CoengageUIProvider.

```tsx
<TextInput /> // -> will render fine during ssr
<TextInput id="not-necessary" /> // -> id is no longer necessary
```

## ColorSchemeProvider

`ColorSchemeProvider` is a new component that will help you manage color scheme ([read full docs](https://coengage.dev/theming/dark-theme/)):

```tsx
import { CoengageUIProvider, ColorSchemeProvider, ColorScheme } from '@coengageui/core';

export default function Demo() {
  const [colorScheme, setColorScheme] = useState('light');
  const toggleColorScheme = (value?: ColorScheme) =>
    setColorScheme(value || (colorScheme === 'dark' ? 'light' : 'dark'));

  return (
    <ColorSchemeProvider colorScheme={colorScheme} toggleColorScheme={toggleColorScheme}>
      <CoengageUIProvider theme={{ colorScheme }}>
        <App />
      </CoengageUIProvider>
    </ColorSchemeProvider>
  );
}

// ...later in other component
import { ActionIcon, useCoengageUIColorScheme } from '@coengageui/core';
import { SunIcon, MoonIcon } from '@modulz/radix-icons';

function Demo() {
  const { colorScheme, toggleColorScheme } = useCoengageUIColorScheme();
  const dark = colorScheme === 'dark';

  return (
    <ActionIcon
      variant="outline"
      color={dark ? 'yellow' : 'blue'}
      onClick={() => toggleColorScheme()}
      title="Toggle color scheme"
    >
      {dark ? (
        <SunIcon style={{ width: 18, height: 18 }} />
      ) : (
        <MoonIcon style={{ width: 18, height: 18 }} />
      )}
    </ActionIcon>
  );
}
```

## Margin props

All components now support setting following props to control margins:

- `m` – sets `margin` property on root element
- `my` – sets `margin-top` and `margin-bottom` properties on root element
- `mx` – sets `margin-right` and `margin-left` properties on root element
- `mt` – sets `margin-top` property on root element
- `mb` – sets `margin-bottom` property on root element
- `ml` – sets `margin-left` property on root element
- `mr` – sets `margin-right` property on root element

```tsx
import { Button, TextInput } from '@coengageui/core';

function Demo() {
  return (
    <div>
      <Button mx={20}>My button</Button>
      <TextInput mt="md" />
    </div>
  );
}
```

## Other changes

- [ActionIcon](https://coengage.dev/core/action-icon/) and [Button](https://coengage.dev/core/button/) components now support new `default` variant
- [Pagination](https://coengage.dev/core/pagination/) component now supports adding first and last controls and removing next/prev buttons
- [SimpleGrid](https://coengage.dev/core/simple-grid/) component now supports `theme.breakpoints` usage in `breakpoints` prop
- [SimpleGrid](https://coengage.dev/core/simple-grid/) was migrated to use CSS Grid Layout instead of flexbox
- [Group](https://coengage.dev/core/group/) component no longer supports Styles API, you can add styles directly to element and children instead
- New [Space](https://coengage.dev/core/space/) component lets you add spacing between elements without directly subscribing to theme
- [Overlay](https://coengage.dev/core/overlay/) and [LoadingOverlay](https://coengage.dev/core/loading-overlay/) components now support `radius` prop
