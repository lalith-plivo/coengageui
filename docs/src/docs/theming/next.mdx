# Usage with Next.js

CoengageUI is fully compatible with Next.js, follow following guide to setup your application

## Get started with template

If you are starting new project you can skip all configuration steps and use [Next.js template](https://github.com/coengagedev/coengageui-next-template).
To get started [follow this link](https://github.com/coengagedev/coengageui-next-template/generate) and create new repository.
Alternatively you can download or clone [repository](https://github.com/coengagedev/coengageui-next-template) to get started on your machine.

## Adding to existing Next.js project

1. Install `@coengageui/next`

```bash
yarn add @coengageui/next

npm install @coengageui/next
```

1. Create `pages/_document.tsx` file:

```tsx
import { createGetInitialProps } from '@coengageui/next';
import Document, { Head, Html, Main, NextScript } from 'next/document';

const getInitialProps = createGetInitialProps();

export default class _Document extends Document {
  static getInitialProps = getInitialProps;

  render() {
    return (
      <Html>
        <Head />
        <body>
          <Main />
          <NextScript />
        </body>
      </Html>
    );
  }
}
```

2. (Optional) Replace your `pages/_app.tsx` with

```tsx
import { AppProps } from 'next/app';
import Head from 'next/head';
import { CoengageUIProvider } from '@coengageui/core';

export default function App(props: AppProps) {
  const { Component, pageProps } = props;

  return (
    <>
      <Head>
        <title>Page title</title>
        <meta name="viewport" content="minimum-scale=1, initial-scale=1, width=device-width" />
      </Head>

      <CoengageUIProvider
        withGlobalStyles
        withNormalizeCSS
        theme={{
          /** Put your coengage theme override here */
          colorScheme: 'light',
        }}
      >
        <Component {...pageProps} />
      </CoengageUIProvider>
    </>
  );
}
```

## Create your own getInitialProps

In some cases you would want to have access to `getInitialProps` in Next Document,
to do so you will need to setup CoengageUI ssr styles extraction on your own:

```tsx
import Document, { DocumentContext } from 'next/document';
import { ServerStyles, createStylesServer } from '@coengageui/next';

const stylesServer = createStylesServer();

export default class _Document extends Document {
  static async getInitialProps(ctx: DocumentContext) {
    const initialProps = await Document.getInitialProps(ctx);

    // Add your app specific logic here

    return {
      ...initialProps,
      styles: [
        initialProps.styles,
        <ServerStyles html={initialProps.html} server={stylesServer} key="styles" />,
      ],
    };
  }
}
```

## CoengageUI components with Next Link

You can use polymorphic components (Button, ActionIcon, Anchor, etc.) with Next Link, to do so:

- set `component` prop to `a`
- wrap it with Link

```tsx
import Link from 'next/link';
import { Button } from '@coengageui/core';

function Demo() {
  return (
    <Link href="/hello" passHref>
      <Button component="a">Next link button</Button>
    </Link>
  );
}
```
